# AGENTS.md — hit-no-recipe 作業ルール

このリポは「ヒットのレシピ」（https://hit-no-recipe.pages.dev/）。
単一ファイルHTML＋vanilla JS、ノービルド構成。アイデア発想ツール。

## ブランチ・PR運用
- **mainへの直接コミット禁止**。必ず作業ブランチを切り、Pull Requestを提出する
- レビューとマージ判定は Claude Code 側が行う。PRの説明に「変更点」「テスト手順」を必ず書く
- `.github/workflows/` とデプロイ設定（Cloudflare Pages）には触らない。push→CI→自動デプロイの構成を変えない

## ブランド分離（重要）
- このサイトは kkpwebninja.com（web忍者の砦）ネットワークと**無関係を保つ**
- 砦へのリンク・GA4・AdSense・アフィリエイトリンク・個人名（作者名）をコードに入れない
- フッターの「© kkp-15 (GitHub) · MIT License・外部送信なし」表記は変更しない

## 保護指定（削除・変更禁止）
- `store` オブジェクトの try-catch フォールバック（localStorage が使えない環境への対策）
- `esc()` による XSS エスケープ。保存データの表示は必ず `esc()` を通す
- モード切替（ふつう／こども）は検討の末に廃止済み。**再実装しない**

## 設計原則
- **外部送信なし**: fetch / XMLHttpRequest / sendBeacon / 外部スクリプト読み込みを追加しない
- 外部依存なし: CDN・ライブラリの追加はPR前に相談（原則vanilla JSで書く）
- 単一 index.html を維持（ファイル分割しない）
- UIテキストは中間トーン（専門用語にはカッコで補足）。装飾的な絵文字は増やさない
- 変更は最小限に。既存の見た目・文言・データ構造を「改善」目的で勝手に変えない
- スマホ（375px幅）でレイアウトが崩れないことを確認してからPRを出す
